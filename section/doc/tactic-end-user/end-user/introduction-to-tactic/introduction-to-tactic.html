TACTIC is an extensible, web-based production asset management system specifically designed to organize and manage assets across large-scale productions and entire organizations. What this means is that TACTIC centralizes the following aspects of prodution

  • Production data management

  • Scheduling

  • File (Asset) Management

  • Communication

  • Reporting

  • and more

What can I do with TACTIC?

TACTIC End-Users have various roles in the projects often no different than what they are without TACTIC. Interactions with the TACTIC interface are often moulded to provide each group of End-User the ideal interface. To put it simple, providing them with exactly what they need to see and how they need to see it.

The table below gives an idea to how various End-Users might interact with TACTIC in a single organization

Artist The Artist often simply needs to see a view of all tasks assigned to them. From these Tasks, users can update a status, checkin and checkout their files, log work hours and communicate with supervisors using notes.
Supervisor/Manager The manager often looks at the project from a scheduling perspective which mainly involves scheduling tasks but, also from a Content (Search Object) angle. Supervisors will often not want to look at the project at just the task level, it is often good to have views where the supervisor can start at the top of the project and dig down through the various relationships getting a better overall view of the project.
Client Clients often need to review preview material (videos, images etc) and delivering this content is often a hurdle. Clients can be provide very simple client views where Tactic can use its built in web technology to deliver files and allow for feedback without FTP or Email.
Executive Executives need a high level view of the project. This often involved building views or "reports" which correlate data from all aspects of the project to provide cost reporting, completion, efficiency and location of bottlenecks.

What can I do in TACTIC?

TACTIC is a transaction system allowing users to search, interact, update and report on project data and files. These interactions are carried out using a few main operations:

  • Search - When in TACTIC one of the first things you will need to do is search for something. Results are often displayed in a list where you can interact with your project data

  • Insert - When starting a new project, you often need to add or "Insert" new data. This is done through various interface tools like forms, multi inserts, CSV import and more

  • Update - Often it is necessary to update existing data. This might be updating a status, adding keywords, changing an assigned user etc. Specific views and tools also allow for selecting multiple items to edit.

  • Checkin - Part of TACTIC's power is the ability to checkin and manage your file assets. When you do a checkin, TACTIC versions and names the directories and files in your file server.

  • Checkout - When you need to work on some files that are already checked into TACTIC, you can check them out. This process ether points you to the files where they are or, downloads them to your computer.

  • Undo/Redo - Everything carried out in TACTIC can be undone. If something goes wrong you can simply roll back.

How does TACTIC work?

TACTIC is simply a transaction based system that uses modern web technologies to bring everything in your organization together seamlessly. There are 2 main types of data that is managed together in TACTIC. These 2 main aspects are also part of what we use to describle TACTIC.

Production - This is the information/lists usually tracked in spreadsheets. It usually lists of what you manage along with task assigned, status and schedule data

Assets - This is usually the actual files themselves. When managing asstes (files), the directory and file naming along with proper version control is what asset management is.

When these two major components are coupled together as an"Production Asset Managent System", it becomes very powerful because the full lifecycle of your asests is actually tracked.

Meta Data Production data stored in the Database The word database itself helps to explain how it works. It is the base location for all of your data. This data is all of the information that is often tracked in spreadsheets, emails, sticky notes, whiteboards etc. What a database provides is a powerful central location for this information which helps keep everything in one place.
File Data Files stored in the File System

Files in Tactic are managed externally in a file system no different than if users saved their files directly. The file server typically has a root 'assets' directory where Tactic handles the directory and filenames for the complete file system.
